# Explotación


### Usuario


Buscando en Google, se determina que el puerto abierto `50051` corresponde con un protocolo llamado _gRPC_. Para más información:


[https://www.paradigmadigital.com/dev/grpc-que-es-como-funciona/](https://www.paradigmadigital.com/dev/grpc-que-es-como-funciona/)


Para interaccionar con dicho protolo se utiliza la herramienta _grpcurl_ para listar los servicios disponibles_._


![Untitled.png](/assets/img/hackthebox/machines/pc/usuario1.png)

Para cada servicio se obtienen los métodos disponibles.


![Untitled.png](/assets/img/hackthebox/machines/pc/usuario2.png)



![Métodos disponibles de SimpleApp](/assets/img/hackthebox/machines/pc/usuario3.png)


Se prueba a ejecutar los servicios obtenidos


![Untitled.png](/assets/img/hackthebox/machines/pc/usuario4.png)


Al intentar crear un usuario y registrarse con él se pueden obtener respuestas del servidor


![Untitled.png](/assets/img/hackthebox/machines/pc/usuario5.png)


Sin embargo para el método _getInfo_ se pide un token del cual no se dispone:


![Untitled.png](/assets/img/hackthebox/machines/pc/usuario6.png)



Lo que se ve al ejecutar el método es el cuerpo de la respuesta, pero el _token_ que se solicita debería estar en las cabeceras. Al utilizar la flag `-vv` se pone el modo verboso de la herramienta y se muestran las cabeceras. Al hacer login con un usuario se puede obtener el token y ejecutar el método _getInfo_.


![Untitled.png](/assets/img/hackthebox/machines/pc/usuario7.png)
